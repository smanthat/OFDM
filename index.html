The System

A full OFDM pipeline that runs in two logical modes from the same waveform — communication mode for bit recovery, and radar mode for target range and velocity estimation.

Bits→ QPSK Map→ 64-SC Map→ IFFT→ Cyclic Prefix→ Channel→ FFT→ Equalize→ Demap→ BER

// Communication Mode

Recovers transmitted bits and measures link quality. Cyclic prefix removal, FFT, QPSK demapping, then a bit-by-bit comparison against the original stream.

Did the receiver recover the transmitted bits correctly?

// Radar Mode

Uses known OFDM symbols to estimate the channel response, builds a range profile via IFFT, then applies a Doppler FFT across symbols to form a range-Doppler map.

Where is the target, and how fast is it moving?

Hardware Validation

The signal chain didn't just pass in simulation — it was validated on real test equipment through an analog baseband loopback on an Analog Discovery 2.

// Loopback Setup

Exported the OFDM waveform's I and Q samples and drove them out the AD2's two waveform-generator channels, then captured the returning analog signals on the AD2 oscilloscope channels.

// Recovery Pipeline

Reconstructed the complex baseband signal and built a Python recovery chain — DC-offset removal, packet-length estimation by autocorrelation, resampling to the reference length, and I/Q-ambiguity resolution — feeding into the OFDM receiver.
